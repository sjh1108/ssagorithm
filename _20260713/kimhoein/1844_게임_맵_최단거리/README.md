# 게임 맵 최단거리

- 문제: [프로그래머스 1844번 - 게임 맵 최단거리](https://school.programmers.co.kr/learn/courses/30/lessons/1844)
- 유형: BFS, 최단 거리
- 코드: [게임_맵_최단거리.java](./게임_맵_최단거리.java)

## 내가 이해한 문제

- 캐릭터는 좌측 상단에서 시작한다.
- 상대 팀 진영은 우측 하단에 있다.
- 맵에서 `1`은 이동 가능한 칸이고, `0`은 벽이다.
- 상하좌우로 한 칸씩 이동할 수 있다.
- 상대 팀 진영까지 도착하는 최단 거리를 구해야 한다.
- 도착할 수 없으면 `-1`을 반환한다.

## 내가 처음 잡은 풀이 방향

- 캐릭터의 현재 위치를 기준으로 BFS를 돌린다.
- BFS를 돌다가 상대 팀 진영에 도착하면 반복을 끝내고 이동 횟수를 반환한다.
- 시작점은 항상 `(0, 0)`이다.
- 큐에 캐릭터의 위치와 이동 횟수를 함께 넣는다.
- 이미 방문한 위치는 다시 방문하지 않는다.

## 모델링 정리

### 상태

- 현재 위치 `r, c`
- 현재까지 지나온 칸 수 `dist`

### 전이

- 현재 위치에서 상하좌우 네 방향으로 한 칸 이동한다.
- 다음 위치는 `nr = r + dr[i]`, `nc = c + dc[i]`이다.
- 이동하면 거리는 `dist + 1`이 된다.

### 제약

- 맵 밖으로 나가면 안 된다.
- `maps[nr][nc] == 0`인 벽으로 이동하면 안 된다.
- 이미 방문한 칸은 다시 방문하지 않는다.

### 목표

- 현재 위치가 `(n - 1, m - 1)`이면 `dist`를 반환한다.
- 큐가 빌 때까지 도착하지 못하면 `-1`을 반환한다.

## 헷갈렸던 부분

- 문제 설명에서는 도착점이 `(n, m)`처럼 보이지만, Java 배열의 인덱스는 `0`부터 시작하므로 도착점은 `(n - 1, m - 1)`이다.
- 배열 범위 체크는 `nx > n`이 아니라 `nx >= n`이어야 한다.
- 시작 칸도 거리로 세기 때문에 시작 노드의 거리는 `1`로 둔다.
- `maps[nr][nc] == 0`인 벽을 반드시 확인해야 한다.
- 큐에 넣을 때 방문 처리해야 같은 위치가 중복으로 들어가지 않는다.

## 제출 코드에 대해

- Java 파일에는 문제를 풀었을 때 직접 작성한 코드를 그대로 남겼다.
- 코드의 동작은 바꾸지 않고 들여쓰기와 공백만 정리했다.

## 이 문제에서 얻은 포인트

- BFS는 모든 이동 비용이 같을 때 최단 거리를 구하기 좋다.
- 격자 문제에서는 문제의 좌표와 배열의 인덱스를 구분해야 한다.
- 배열 길이가 `n`이면 가능한 인덱스는 `0`부터 `n - 1`까지다.
- BFS에서 처음 방문한 위치는 최단 거리로 방문한 것이므로 다시 방문할 필요가 없다.
