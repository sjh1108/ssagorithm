# 상태이상이 있는 자동 전투 — 풀이 (의도와 목적)

## 문제 핵심
B1의 전투에 **독·기절·보호막** 세 가지 상태이상이 추가됩니다. 전사는 공격 대상을 기절시키고, 궁수는 중독시키며, 성직자는 아군에 보호막을 겁니다. 라운드 종료 시 독 피해가 들어가고, 보호막은 모든 피해를 먼저 흡수합니다.

## 어떻게 접근했는가 — 상속이 아니라 "붙이는" 컴포넌트
"독 걸린 유닛", "기절한 유닛"을 상속으로 표현하려 하면 조합이 폭발합니다(독+기절, 독+보호막, …). 상태는 **동시에 여러 개** 걸릴 수 있고 **켜졌다 꺼지는** 성질이라, 타입 계층으로 다룰 대상이 아닙니다.

핵심 관찰:

> 상태이상은 유닛의 "종류"가 아니라 유닛에 **부착되는 상태 묶음**이다. 그러니 별도 `Effects` 객체를 만들어 각 유닛이 **하나씩 소유(합성)**하게 한다.

이렇게 하면 병종 계층(B1의 다형성)은 그대로 두고, 직교하는 축으로 상태를 얹을 수 있습니다.

## 왜 이렇게 설계·구현했는가
`Effects` 컴포넌트가 보호막·독(잔여·피해량)·기절(잔여)을 들고, **피해 흡수 규칙**까지 자기 안에 캡슐화합니다.

```java
static class Effects {
    int shield, poisonTurns, poisonDmg, stunTurns;
    int takeDamage(int amount) {          // 보호막으로 먼저 흡수, 체력에 갈 잔여 반환
        if (shield > 0) { int a = Math.min(shield, amount); shield -= a; amount -= a; }
        return amount;
    }
}
```

유닛의 `hit()`은 항상 `eff.takeDamage()`를 거치므로, **모든 피해가 자동으로 보호막을 먼저 깎습니다**(공격이든 독이든). 병종 `act()`는 B1에서 각자 "상태 부여" 한 줄씩만 늘어납니다(전사 → `stunTurns=1`, 궁수 → 독 세팅).

라운드 구조에 두 가지가 추가됩니다.
- **자기 차례 진입 시** 기절 검사: 잔여>0이면 1 깎고 스킵.
- **라운드 종료 시** 독 처리: 번호 순으로 독 피해 적용 후 잔여 1 감소.

```java
for (Unit u : order) {
    if (u.hp <= 0) continue;
    if (u.eff.stunTurns > 0) { u.eff.stunTurns--; continue; }  // 기절 → 행동 스킵
    u.act(this);
    if (finish() != -1) return ...;
}
for (Unit u : units)                       // 라운드 끝: 독 (번호 순)
    if (u.hp > 0 && u.eff.poisonTurns > 0) { u.hit(u.eff.poisonDmg); u.eff.poisonTurns--; }
```

## 놓치기 쉬운 포인트
- **"방어 무시 ≠ 보호막 관통"**: 궁수의 방어 무시는 *방어력 스탯*만 무시하지, 보호막은 못 뚫습니다. 흡수 로직을 우회시키면 안 됩니다.
- **독은 라운드 종료 시, 번호 순**으로 처리해야 결과가 유일해집니다. 이때 **양 팀이 동시에 전멸**할 수 있어(상호 독사) 무승부 판정이 필요합니다 — 그래서 `finish()`가 "둘 다 죽음 → 무승부"를 따로 반환합니다.
- **상태는 갱신(refresh)이지 누적이 아닙니다**: 재중독/재기절 시 잔여를 다시 세팅할 뿐 더하지 않습니다.
- 성직자는 "보호막 없는 아군" 중 비율 최저부터 보호막을 걸고, 모두 보호막이 있으면 치유로 전환합니다.

## 복잡도
B1과 동일하게 **O(R · n²)** — 라운드 종료 독 처리는 라운드당 O(n)으로 지배항에 묻힙니다.
